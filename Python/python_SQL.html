Section 1: Understanding Database Systems

The ability to store, manage, and retrieve data efficiently and reliably is a cornerstone of modern computing. This section explores the evolution from simple file-based storage to sophisticated Database Management Systems (DBMS), laying the groundwork for understanding the relational model that forms the core of the CBSE Class 12 syllabus.

1.1 From File Systems to Databases: The 'Why'

Before the advent of databases, data was commonly stored in individual files managed directly by the operating system. This approach, known as the file processing system, was adequate for simple applications but proved deeply problematic as data volumes and user requirements grew. This system was plagued by several critical limitations that necessitated a more structured solution.

  • Data Redundancy: This refers to the unnecessary duplication of data. In a file system, different applications or departments within an organization often maintained their own separate data files. This duplication wastes storage space and leads to other problems.
  • Data Inconsistency: This is a direct consequence of data redundancy. If a student's address changes, it might be updated in one file but not another. The system now holds conflicting information, creating a state of data inconsistency.
  • Complex Data Access: Retrieving data from a file system required writing a custom program for every new type of query. This was inflexible and slow.
  • Lack of Security: File systems typically offer password protection at the file level but lack granular control. It was difficult to enforce security policies where different users have different access rights to the same data.
  • No Concurrent Access Control: When multiple users need to access and modify the same data simultaneously, a file system offers no mechanism to prevent conflicts, which can lead to data corruption.

1.2 Introduction to Database Management Systems (DBMS)

A Database Management System (DBMS) is the software solution designed to overcome the limitations of file systems. It is a software package that allows users to create, define, maintain, and control access to a database. A database itself is a systematic and organized collection of logically related data. By centralizing data storage and management, a DBMS provides numerous advantages like minimizing redundancy, ensuring consistency, improving security, and providing data independence.

1.3 The Relational Model (RDBMS): A Structured Approach

An RDBMS is a DBMS based on the relational model. The fundamental principle of this model is that all data is stored in a simple, intuitive, and structured format: two-dimensional tables, formally known as relations.

Section 2: Anatomy of a Relational Database

2.1 Core Terminology

  • Relation: The formal, mathematical term for a table.
  • Tuple: The formal term for a row or a record in a relation.
  • Attribute: The formal term for a column or a field in a relation.
  • Domain: The set or pool of all permissible, atomic values for a given attribute.

2.2 Structural Properties

  • Degree: The total number of attributes (columns) in a relation.
  • Cardinality: The total number of tuples (rows) in a relation. The SQL command SELECT COUNT(*) FROM table_name; calculates the cardinality.

2.3 The Role of Keys

Keys are attributes that uniquely identify each tuple and establish links between relations.

Section 3: Introduction to SQL

3.1 The Language of Databases

  • DDL (Data Definition Language): Defines and manages the database structure. Commands: CREATE, ALTER, DROP.
  • DML (Data Manipulation Language): Manages the data within tables. Commands: SELECT, INSERT, UPDATE, DELETE.

3.2 Essential SQL Data Types

  • CHAR(n): Fixed-length character string. Padded with spaces if shorter than n.
  • VARCHAR(n): Variable-length character string. More space-efficient.
  • INT / INTEGER: Whole numbers.
  • FLOAT / DECIMAL(p, s): Numbers with decimal points.
  • DATE: Stores date values, typically in 'YYYY-MM-DD' format.

Section 4: Data Definition Language (DDL)

4.1 CREATE Command

Used to build new database objects.

-- Creating a Database
CREATE DATABASE School;

-- Creating a Table
CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Name VARCHAR(50) NOT NULL,
    City VARCHAR(30),
    DOB DATE
);

4.2 ALTER Command

Modifies the structure of an existing table.

-- Adding a Column
ALTER TABLE Student ADD Grade CHAR(1);

-- Modifying a Column
ALTER TABLE Student MODIFY COLUMN City VARCHAR(50);

-- Dropping a Column
ALTER TABLE Student DROP COLUMN Grade;

4.3 DROP Command

Permanently deletes database objects.

-- Dropping a Table
DROP TABLE Student;

-- Dropping a Database
DROP DATABASE School;

4.4 SQL Constraints

Rules enforced on data columns.

  • NOT NULL: Column cannot have a NULL value.
  • UNIQUE: All values in a column must be different.
  • PRIMARY KEY: Uniquely identifies each record.
  • FOREIGN KEY: Links to a primary key in another table.
  • CHECK: Ensures values satisfy a specific condition (e.g., Age INT CHECK (Age >= 18)).
  • DEFAULT: Provides a default value for a column.

Section 5: Data Manipulation Language (DML)

5.1 INSERT INTO Command

-- Syntax 1 (Recommended)
INSERT INTO Student (RollNo, Name, City)
VALUES (101, 'Arun Sharma', 'Delhi');

-- Syntax 2 (Requires values for all columns in order)
INSERT INTO Student
VALUES (102, 'Bhavna Singh', 'Mumbai', '2005-08-15');

5.2 UPDATE Command

Crucially, always use a WHERE clause to avoid updating all rows.

UPDATE Student
SET City = 'Chennai'
WHERE RollNo = 101;

5.3 DELETE Command

Crucially, always use a WHERE clause to avoid deleting all rows.

DELETE FROM Student
WHERE RollNo = 102;

Section 6: The SELECT Statement

The SELECT statement is used to query data from tables.

6.1 Basic Retrieval

-- Select specific columns
SELECT Name, City FROM Student;

-- Select all columns
SELECT * FROM Student;

6.2 Filtering with WHERE

Clauses for filtering: =, !=, >, <, AND, OR, BETWEEN, IN, IS NULL, LIKE.

SELECT Name FROM Student WHERE City = 'Delhi';
SELECT Name FROM Student WHERE Name LIKE 'A%';

6.3 Sorting with ORDER BY

Sorts results. ASC (default) for ascending, DESC for descending.

SELECT Name, Marks FROM Student ORDER BY Marks DESC;

Section 7: Summarizing and Grouping Data

7.1 Aggregate Functions

Perform calculations across a set of rows. Includes COUNT(), SUM(), AVG(), MAX(), MIN().

SELECT COUNT(DISTINCT City) FROM Student;

7.2 Organizing Data with GROUP BY

Groups rows that have the same values into summary rows.

SELECT City, COUNT(*) FROM Student GROUP BY City;

7.3 Filtering Groups with HAVING

Filters groups created by GROUP BY. WHERE filters rows before grouping; HAVING filters groups after.

SELECT City, COUNT(*)
FROM Student
GROUP BY City
HAVING COUNT(*) > 10;

Section 8: Joining Tables

Joins combine rows from two or more tables based on a related column.

8.2 Equi Join

Uses an equality operator in the join condition.

SELECT Student.Name, Course.CourseName
FROM Student
INNER JOIN Course ON Student.RollNo = Course.RollNo;

8.3 Natural Join

Automatically joins tables based on all columns with the same name. Use with caution.

SELECT Name, CourseName
FROM Student
NATURAL JOIN Course;

Section 9: The Python-SQL Interface

Connect a Python program to a SQL database to perform database operations programmatically.

Workflow Steps:

  1. Import the connector library (e.g., mysql.connector).
  2. Establish a connection using connect().
  3. Create a cursor object from the connection.
  4. Execute SQL queries using cursor.execute().
  5. Fetch results with fetchone(), fetchall(), or fetchmany().
  6. Commit transactions for DML commands using connection.commit().
  7. Close the cursor and connection.

Example Program:

import mysql.connector

try:
    db_connection = mysql.connector.connect(
        host="localhost",
        user="your_username",
        password="your_password",
        database="School"
    )
    db_cursor = db_connection.cursor()
    query = "SELECT RollNo, Name FROM Student WHERE Grade = 'A';"
    db_cursor.execute(query)
    student_records = db_cursor.fetchall()
    for row in student_records:
        print(f"Roll No: {row[0]}, Name: {row[1]}")
except mysql.connector.Error as e:
    print(f"Error: {e}")
finally:
    if db_cursor:
        db_cursor.close()
    if db_connection and db_connection.is_connected():
        db_connection.close()
        print("MySQL connection is closed")

Section 10: CBSE Board Exam Question Bank

Category 1: Very Short Answer Questions (1 Mark)

  1. The number of tuples in a relation is called its __________.
    Answer: Cardinality
  2. Which of the following is a DDL command?
    • (a) SELECT
    • (b) UPDATE
    • (c) ALTER
    • (d) INSERT
    Answer: (c) ALTER
  3. State True or False: A primary key can have a NULL value.
    Answer: False

Category 2: Short Answer Questions (2-3 Marks)

  1. Differentiate between the CHAR and VARCHAR data types in SQL.
    The CHAR data type is used to store fixed-length character strings. If the input string is shorter than the defined length, it is padded with spaces. The VARCHAR data type stores variable-length strings, which is more space-efficient for data of varying lengths.
  2. What is the difference between the DELETE and DROP TABLE commands in SQL?
    The DELETE command is a DML command used to remove rows from a table, but it leaves the table structure intact. The DROP TABLE command is a DDL command that permanently removes the entire table, including its structure and data.

Category 3: Long Answer & Case-Study Questions (4-5 Marks)

  1. Case Study 1: Consider tables DOCTOR(DID, DName, Dept, Experience) and SALARY(DID, Basic, Allowance, Gender). Write SQL queries for the following:
    • (i) Display the names of all doctors who are in the 'Medicine' department and have more than 10 years of experience.
    • (ii) Display the name of each department and the number of doctors in it.
    • (iii) Display the department names that have more than one doctor.

    (i) SELECT DName FROM DOCTOR WHERE Dept = 'Medicine' AND Experience > 10;

    (ii) SELECT Dept, COUNT(*) FROM DOCTOR GROUP BY Dept;

    (iii) SELECT Dept FROM DOCTOR GROUP BY Dept HAVING COUNT(*) > 1;
