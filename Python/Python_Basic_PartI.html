1. Introduction to Python

A Brief History: The Story Behind the Name

Python's story begins in the late 1980s with a Dutch programmer named Guido van Rossum. He wanted to create a new scripting language that would be simple, intuitive, and powerful. As a big fan of the British comedy troupe Monty Python, he named his new language "Python" in honor of their TV show, Monty Python's Flying Circus. The first version was released in 1991.

Why is Python a Great First Language?

For students stepping into the world of programming, Python is an excellent choice. Its core philosophy emphasizes code readability and simplicity, allowing you to focus on problem-solving logic rather than complex syntax. Python is used by some of the biggest tech companies in the world, including Google, Netflix, and Instagram.

Your First Program: "Hello, World!"

The first program anyone writes in a new language is one that simply prints "Hello, World!" to the screen. This confirms your setup is correct.

# Our very first Python program!
# The print() function displays text on the screen.
print("Hello, World!")

2. Features of Python

What makes Python so special? Its popularity stems from a combination of powerful features that make it both easy for beginners and robust for experts. Understanding these features helps you appreciate why it's such a well-loved language.

Easy to Learn & Read

Python's syntax is designed to be clean and uncluttered. It uses plain English keywords and indentation instead of curly braces, which makes the code look neat and easy to understand, even for someone who has never coded before. This readability is a core principle of the language.

Interpreted Language

Python is an interpreted language. This means that the code is executed line by line, directly by an interpreter. If there's an error in your code, the program will stop at that line and report it. This makes debugging (the process of finding and fixing errors) much easier for beginners.

Dynamically Typed

This is a powerful feature that simplifies coding. In many languages, you must declare the type of a variable before you can use it. In Python, you don't have to. Python automatically determines the data type of a variable at the time you assign a value to it.

Cross-Platform and Portable

Python code is portable. This means you can write a program on a Windows computer, and it will run on a Mac, a Linux machine, or even a Raspberry Pi without needing any changes. This "write once, run anywhere" capability is a huge advantage.

Free and Open-Source

Python is completely free to use, download, and distribute. It is developed under an open-source license, which means its source code is publicly available. Anyone can contribute to its development, and this has led to a massive, active community that constantly improves the language.

"Batteries Included" Philosophy

Python is often described as a "batteries included" language. This refers to its huge Standard Library, which is a collection of pre-written code (called modules) that you can use in your own programs, saving you from having to write everything from scratch.

3. Python IDE and Execution

To start writing and running Python programs, you need a coding environment. While you can use a simple text editor, it's much more efficient to use an Integrated Development Environment (IDE).

What is an IDE?

Think of an IDE as a programmer's workshop. It's a software application that bundles all the essential tools for programming into one place. A typical IDE includes:

  • A Source Code Editor: A text editor designed for writing code, often with features like syntax highlighting to make code more readable.
  • A Build Automation Tool: For languages that need compiling, this automates the process. For Python, this is less critical, but the IDE still manages the execution.
  • A Debugger: A tool that helps you find and fix errors in your code by allowing you to run it step-by-step.

Meet IDLE: Your First Python IDE

When you install Python, it comes with its own simple IDE called IDLE (Integrated Development and Learning Environment). As the name suggests, it's specifically designed for beginners and is a great place to start your Python journey. It's lightweight, easy to use, and available on all platforms where Python is installed.

Installation Guide (for Windows)

  1. Visit the Official Website: Open your web browser and go to the official Python downloads page: https://www.python.org/downloads/.
  2. Download the Installer: The website will automatically detect your operating system and suggest the latest stable version. Click the button that says "Download Python 3.x.x".
  3. Run the Installer: Once the download is complete, find the .exe file in your Downloads folder and double-click it.
  4. Crucial Step - Add to PATH: Before you click "Install Now," look at the bottom of the window. It is very important that you check the box that says "Add Python 3.x to PATH". This allows you to run Python from your computer's command line.
  5. Complete Installation: Click "Install Now" and follow the on-screen instructions. Once it's done, you have successfully installed Python and IDLE!

Execution Modes in Python

The CBSE syllabus requires you to be familiar with Python's two main execution modes: Interactive Mode and Script Mode. IDLE provides easy access to both.

Interactive Mode (The Shell)

When you first open IDLE, you are greeted by a window called the Python Shell with a >>> prompt. This is Interactive Mode. Think of it as a direct conversation with the Python interpreter. You type one command, press Enter, and Python immediately executes it and shows you the result. It's perfect for testing small snippets of code or checking variable values.

Let's try calculating the area of a rectangle:

>>> length = 10
>>> width = 5
>>> area = length * width
>>> print(area)
50
>>>

Script Mode (The Editor)

While Interactive Mode is great for quick tests, you can't save your work. For writing longer, more complex programs, you use Script Mode. A script is simply a text file where you write your Python code, saved with a .py extension.

To use Script Mode in IDLE:

  1. In the Shell window, go to File > New File.
  2. A new, blank window will open. This is the Editor.
  3. Write your complete program here.
  4. Save the file (File > Save). Give it a name ending in .py.
  5. To run your program, go to Run > Run Module (or press the F5 key).

Let's write our rectangle area calculator as a reusable script:

# File: rectangle_area.py
# This program calculates and prints the area of a rectangle.

length = 10
width = 5
area = length * width

print("The length of the rectangle is:", length)
print("The width of the rectangle is:", width)
print("The area of the rectangle is:", area)

When you run this script, the Shell window will show:

The length of the rectangle is: 10
The width of the rectangle is: 5
The area of the rectangle is: 40

4. Tokens, Identifiers, and Literals

What are Tokens?

Just like a human language is made of sentences, words, and punctuation, a programming language is made of statements, tokens, and punctuators. A token is the smallest individual unit in a Python program. The interpreter breaks down your code into these tokens to understand it. Think of them as the "words" and "punctuation marks" of the Python language.

The main types of tokens in Python are:

  • Keywords: Reserved words with special meanings.
  • Identifiers: Names you give to things like variables.
  • Literals: Fixed values like numbers or text.
  • Operators: Symbols that perform operations, like + or *.
  • Punctuators: Symbols that organize the code structure, like (), or #.

Let's break down a simple line of code to see the tokens:

student_age = 16 # Assigning age
  • student_age: This is an Identifier.
  • =: This is an Operator.
  • 16: This is a numeric Literal.
  • #: This is a Punctuator.
  • Assigning age: This is a comment, which is ignored by the interpreter but is made up of its own tokens.

Keywords

Keywords are the reserved words of the Python language. They have predefined, special meanings and are used to define the structure and syntax of your program. You cannot use a keyword as a name for your variables, functions, or any other identifier. Python keywords are case-sensitive and are always written in lowercase, except for True, False, and None.

Identifiers

An Identifier is a name that you, the programmer, give to an entity in your program. This could be the name of a variable, a function, or a class. Choosing good, descriptive identifier names (e.g., student_name instead of s) makes your code much easier to read and understand.

Rules for Naming Identifiers in Python:

  • An identifier must start with a letter (a-z, A-Z) or an underscore (_). It cannot start with a number.
  • The rest of the name can consist of letters, numbers (0-9), and underscores.
  • No other special characters (like !, @, #, $, %) are allowed.
  • Python is case-sensitive. This means age, Age, and AGE are three different identifiers.
  • An identifier cannot be a Python keyword.

Literals

A Literal is a piece of data whose value is fixed. In other words, literals are the actual constant values you use in your code. When you write x = 10, 10 is an integer literal.

Python has several types of literals:

  • String Literals: A sequence of characters enclosed in quotes.
    • 'Hello' (single quotes)
    • "Python" (double quotes)
    • """This is a multi-line string.""" (triple quotes)
  • Numeric Literals: These include integers, floating-point numbers, and complex numbers.
    • Integer: 101, -50, 0
    • Float: 12.5, -3.14
    • Complex: 3+4j
  • Boolean Literals: Used to represent truth values. There are only two: True and False.
  • Special Literal: Python has one special literal, None, which is used to signify the absence of a value.
  • Collection Literals: Python also has literals for creating data collections, like lists and tuples (e.g., [10, 20, 30]).

5. Data Types and Variables

In programming, we constantly work with data—numbers, text, true/false values, and more. A data type is a classification that tells the computer what kind of data a value is and what operations can be performed on it. A variable is a name we use to store and refer to that data in the computer's memory.

Variables as Labels or References

In some programming languages, a variable is like a box where you put a value. In Python, it's more helpful to think of a variable as a label or a post-it note that you stick onto a value. The value itself is an "object" that exists in the computer's memory. When you write x = 100, you are not putting 100 inside a box named x. Instead, you are creating an integer object with the value 100 and placing the label 'x' on it.

This distinction is subtle but incredibly important for understanding how Python works, especially with more complex data types. Let's trace this concept:

  1. x = 10: Python creates an integer object 10 in memory. The label x now points to this object.
  2. y = x: Python does not create a new object. It simply takes another label, y, and sticks it onto the same 10 object that x is pointing to. Now, both x and y refer to the same value.
  3. x = 20: Python creates a new integer object 20. It then moves the label x from the 10 object to this new 20 object. The label y is unaffected and remains pointing to the original 10 object.

This "labeling" model explains why Python is dynamically typed. You can easily move a label from an integer object to a string object without any issue.

The Assignment Operator (=)

The equal sign (=) in Python is the assignment operator. It works from right to left, meaning it evaluates the expression on the right and assigns that value to the variable on the left. The statement variable_name = value means: "Evaluate the expression on the right, create an object for that value, and then make the variable name on the left refer to that object."

Core Data Types in Python

  • Numeric Types:
    • Integer (int): Represents positive or negative whole numbers without any decimal part (e.g., 10, -250, 0).
    • Float (float): Represents real numbers with a decimal point (e.g., 3.14, -0.001, 25.0).
  • String (str): Represents a sequence of characters used for textual data. You create them by enclosing characters in single ('), double ("), or triple (""") quotes.
  • Boolean (bool): Represents one of two truth values: True or False. Note that these must be capitalized.

You can check the data type of any variable or value using the built-in type() function.

>>> num = 42
>>> pi = 3.14
>>> name = "Amit"
>>> is_student = True

>>> type(num)
<class 'int'>

>>> type(pi)
<class 'float'>

>>> type(name)
<class 'str'>

>>> type(is_student)
<class 'bool'>

Mutable vs. Immutable Types

This is a critical concept. It relates directly to whether a value can be changed after it's created.

  • Immutable Types: Objects of these types cannot be changed after they are created. If you try to "modify" an immutable object, Python will create a new object with the new value and make your variable point to it. Think of an immutable object like a printed sign. You can't just erase a letter; to change it, you have to create a whole new sign.

    The core data types we've discussed so far—int, float, bool, and str—are all immutable. Tuples are also immutable.

    # String (Immutable)
my_string = "hello"
# Let's try to change the first character to 'H'
# my_string[0] = 'H'  # This will cause a TypeError!

# To "change" the string, we must create a new one.
my_string = "H" + my_string[1:]
print(my_string)  # Output: Hello
  • Mutable Types: Objects of these types can be changed in-place after they are created, without creating a new object. Think of a mutable object like a whiteboard. You can easily erase a part of it and write something new without needing a new whiteboard.

    The primary mutable types you will learn about are lists and dictionaries.

    # List (Mutable)
my_list = [10, 20, 30]
# Let's change the first element to 100
my_list[0] = 100  # This works perfectly!
print(my_list)  # Output: [100, 20, 30]

Understanding this difference is key to avoiding unexpected behavior in your programs.

Identity Operators

Identity operators are used to compare the memory locations of two objects. They check if two operands refer to the exact same object in memory, not just if they have the same value. Python has two identity operators: is and is not.

  • is: Returns True if both operands refer to the same object.
  • is not: Returns True if both operands do not refer to the same object.

Think of it like this: if you have two variables, a and b, and you ask a is b, you're not asking if a and b contain the same value, but if a and b are literally pointing to the exact same piece of data in your computer's memory.

Example:

# Case 1: Same value, same object (for small integers and pre-allocated strings)
x = 10
y = 10
print(x is y)  # Output: True (Python often reuses memory for small integers)
print(x is not y) # Output: False

a = "hello"
b = "hello"
print(a is b)  # Output: True (Python often reuses memory for identical string literals)

# Case 2: Same value, different objects (for lists and larger/dynamically created objects)
list1 = [1, 2, 3]
list2 = [1, 2, 3]
print(list1 is list2)  # Output: False (Even though values are same, they are different list objects)
print(list1 is not list2) # Output: True

# list3 now points to the same object as list1
list3 = list1 
print(list1 is list3) # Output: True

# Case 3: Comparing with None
# None is a special literal in Python, and there is only one instance of None.
# So, when you compare something to None, you should always use 'is' or 'is not'.
my_variable = None
print(my_variable is None) # Output: True
print(my_variable == None) # Output: True (This also works, but 'is' is idiomatic for None)

Key Distinction:

  • == (Equality Operator): Compares the values of two objects. It checks if they have the same content.
  • is (Identity Operator): Compares the identities of two objects. It checks if they are the exact same object in memory.

Understanding this difference is crucial, especially when working with mutable data types like lists, where two separate lists can have the same content but be distinct objects in memory.

Membership Operators

Membership operators are used to test whether a value or variable is found in a sequence (such as a string, list, or tuple). They return True if the value is found in the sequence, and False otherwise. Python has two membership operators:

  • in: Returns True if a value is present in the sequence.
  • not in: Returns True if a value is not present in the sequence.

Syntax: value in sequence or value not in sequence

Example:

# Checking for membership in a string
message = "Hello Python"
print('H' in message)      # Output: True
print('x' in message)      # Output: False
print('Py' not in message) # Output: False (because 'Py' is in message)

# Checking for membership in a list
numbers = [10, 20, 30, 40, 50]
print(30 in numbers)     # Output: True
print(99 in numbers)     # Output: False
print(5 not in numbers) # Output: True

# Checking for membership in a tuple
fruits = ("apple", "banana", "cherry")
print("apple" in fruits) # Output: True
print("grape" not in fruits) # Output: True

Membership operators are very useful for checking for the existence of elements within collections of data.

How to Read Complex Expressions

Students frequently misunderstand how Python evaluates expressions containing both and and or operators. This is because and has a higher precedence than or. This means Python evaluates and operations before or operations within the same expression, much like multiplication is evaluated before addition in mathematics.

Example without Parentheses:

result = True or False and False

A common mistake is to read this from left to right. However, Python's order of operations is different:

  1. Python first evaluates False and False because and has higher precedence. This results in False.
  2. The expression then simplifies to True or False.
  3. Finally, True or False evaluates to True.

Result: True

Consider the same example with parentheses:

result = (True or False) and False

Now, the evaluation proceeds as you might initially expect:

  1. The expression inside the parentheses, (True or False), is evaluated first, which results in True.
  2. The expression then becomes True and False.
  3. Finally, True and False correctly evaluates to False.

Result: False

Using parentheses makes your code more robust by explicitly defining the order of operations, preventing unexpected results, and significantly improving readability for anyone who needs to understand or maintain your code.
