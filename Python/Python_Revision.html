Solved Problems

Conceptual Questions

  1. What is indexing in context to Python strings? Why is it also called two-way indexing?
    In Python strings, each individual character is given a location number, called an index. This process is called indexing. Python allocates indices in two directions:
    • In the forward direction, the indexes are numbered as 0, 1, 2,.... length-1.
    • In the backward direction, the indexes are numbered as -1, -2, -3.... -length.
    This ability to access elements from the beginning (positive index) or the end (negative index) is why it's called two-way indexing.
  2. What is a string slice? How is it useful?
    A sub-part or a slice of a string, say s, can be obtained using s[n:m] where n and m are integers. Python returns all the characters at indices n, n+1, n+2...m-1. For example, 'Well done'[1:4] will give 'ell'. It's useful for extracting substrings.
  3. How are lists different from strings when both are sequences?
    The main differences are:
    • Mutability: Lists are mutable (can be changed), while strings are immutable (cannot be changed).
    • Storage: In consecutive memory locations, a string stores individual characters, while a list stores references to its elements (which can be of any type and stored anywhere in memory).
    • Element Types: Strings store only characters, while lists can store elements of different data types.
  4. What are nested lists?
    When a list is contained in another list as an element, it is called a nested list. For example, a = [2, 3, [4, 5]] is a nested list where the third element is another list.
  5. What are the most appropriate list methods for the following tasks?
    • (a) Delete a given element from the list.
    • (b) Delete 3rd element from the list.
    • (c) Add an element in the end of the list.
    • (d) Add an element in the beginning of the list.
    • (e) Add elements of a list in the end of a list.
    (a) remove()
    (b) pop(2) or del list[2]
    (c) append()
    (d) insert(0, element)
    (e) extend()
  6. How are tuples different from lists when both are sequences?
    The primary differences are:
    • Mutability: Tuples are immutable (cannot be changed after creation), while lists are mutable.
    • Flexibility: Because they are mutable, lists can grow or shrink (add/remove elements), while tuples have a fixed size.
  7. How can you say that a tuple is an ordered list of objects?
    A tuple is an ordered collection of objects. This is evidenced by the fact that the objects can be accessed through the use of an ordinal index, and for a given index, the same element is returned every time. The order of elements is preserved.
  8. The following code is trying to create a tuple with a single item, but it gives an error. Why? What is the solution? 
    >>> t = (6)
>>> len(t)
TypeError: object of type 'int' has no len()
    The expression (6) is evaluated as just the integer 6. To create a tuple with a single element, you must include a trailing comma.
    Correction: t = (6,)
  9. What is the length of the tuple shown below? 
    t = ((((a',1),'b','c'),'d',2),'e',3)
    The length of this tuple is 3. The three elements are:
    • ((('a', 1), 'b', 'c'), 'd', 2) (a nested tuple)
    • 'e' (a string)
    • 3 (an integer)
  10. Can tuples be nested?
    Yes, tuples can contain other compound objects, including lists, dictionaries, and other tuples. Hence, tuples can be nested.
  11. How are dictionaries different from lists?
    Dictionaries are different from lists in that lists are sequential (ordered) collections accessed by an integer index, while dictionaries are collections of key-value pairs accessed by a unique, immutable key.
  12. How are objects stored in lists and dictionaries different?
    In lists, objects (elements) are stored in a specific sequence and accessed via their integer index. In dictionaries, objects (values) are stored in an unordered manner (prior to Python 3.7) and are associated with a unique key. You look up a value using its key, not its position.
  13. When are dictionaries more useful than lists?
    Dictionaries are more useful when you need to store and retrieve data based on a specific identifier (the key) rather than a position. For example, storing phone numbers indexed by a friend's name is much more efficient with a dictionary than searching through a list of (name, number) pairs.
  14. Can sequence operations such as slicing and concatenation be applied to dictionaries? Why?
    No. Sequence operations like slicing and concatenation cannot be applied to dictionaries because they are not sequences. They do not maintain a specific order that these operations rely on.
  15. Why can't Lists be used as keys in a dictionary?
    Lists cannot be used as dictionary keys because keys must be of an immutable type. Since lists are mutable (their contents can change), they cannot be used as keys.
  16. If the addition of a new key:value pair causes the size of the dictionary to grow beyond its original size, an error occurs. True or false?
    Answer: False. Dictionaries are mutable types, meaning they can grow or shrink as needed when elements are added or removed.

  13. Find the error in the following code. State the reason. 
    aLst = {'a':1, 'b':2, 'c':3}
print(aLst['a', 'b'])
    This code will produce a KeyError. Dictionaries are accessed by a single key, not a tuple of keys like ('a', 'b') unless that exact tuple was used as a key. To print the values for keys 'a' and 'b', you would need to access them separately: print(aLst['a'], aLst['b']).
  14. Find the error in the following code fragment. State the reason.

  2. Write a program that reads a string and checks whether it is a palindrome.

# .lower() makes the check case-insensitive.

    print(f"'{string}' is a palindrome.")

  3. Consider a dictionary my_points = {'a':(4,3), 'b':(1,2), 'c':(5,1)}. Write a program to calculate the maximum value for each coordinate index across all points. 
    my_points = {'a': (4, 3), 'b': (1, 2), 'c': (5, 1)}
# Assuming all tuples have the same length (2 in this case)
num_coordinates = len(next(iter(my_points.values())))
for i in range(num_coordinates):
    # Extract all values at the current index 'i'
    values_at_index = [point[i] for point in my_points.values()]
    # Find the maximum value
    max_value = max(values_at_index)
    print(f"Maximum Value at index(my_points, {i}) = {max_value}")
