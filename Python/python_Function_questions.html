Assertion and Reasoning

  1. Assertion: A function is a subprogram.
    Reason: A function exists within a program and works within it when called.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  2. Assertion: Non-default arguments cannot follow default arguments in a function call.
    Reason: A function call can have different types of arguments.
    Solution: (b) Both A and R are true but R is not the correct explanation of A. (The reason is a true statement but doesn't explain the assertion. The correct explanation is that positional arguments must come before keyword arguments.)
  3. Assertion: A parameter having a default in the function header becomes optional in the function call.
    Reason: A function call may or may not have values for default arguments.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  4. Assertion (A): Positional arguments in Python functions must be passed in the exact order in which they are defined in the function signature.
    Reasoning (R): This is because Python functions automatically assign default values to positional arguments.
    Solution: (c) A is true but R is false. (Python does not automatically assign default values to positional arguments; they are required.)
  5. Assertion: A variable declared inside a function cannot be used outside it.
    Reason: A variable created inside a function has a function scope.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  6. Assertion: A variable not declared inside a function can still be used inside the function if it is declared at a higher scope level.
    Reason: Python resolves a name using the LEGB rule where it checks in Local, Enclosing, Global, and Built-in scopes, in the given order.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  7. Assertion: A parameter having a default value in the function header is known as a default parameter.
    Reason: The default values for parameters are considered only if no value is provided for that parameter in the function call statement.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  8. Assertion: A function declaring a variable having the same name as a global variable cannot use that global variable.
    Reason: A local variable having the same name as a global variable hides the global variable in its function.
    Solution: (a) Both A and R are true and R is the correct explanation of A.
  9. Assertion: If the arguments in a function call statement match the number and order of arguments as defined in the function definition, such arguments are called positional arguments.
    Reasoning: During a function call, the argument list first contains default argument(s) followed by positional argument(s).
    Solution: (c) A is true but R is false. (The reasoning is incorrect; positional arguments must come before any keyword/default arguments in a function call).
  10. Assertion (A): To use a function from a particular module, we need to import the module.
    Reason (R): An import statement can be written anywhere in the program before using a function from that module.
    Solution: (b) Both A and R are true but R is not the correct explanation of A. (While both statements are true, the reason doesn't fully explain the assertion. The assertion is true because importing brings the module's definitions into the current namespace.)

Programming Questions

  1. Write a program to input n numbers in a list. Remove the duplicate values and print a fresh list. 
    def remove_duplicate(lst):
    freq = {}  # Dictionary to count occurrences
    update_lst = []

    # Count occurrences of each element
    for i in lst:
        if i in freq:
            freq[i] += 1
        else:
            freq[i] = 1

    # Add elements that appear only once
    for i in lst:
        if freq[i] == 1:
            update_lst.append(i)

    return update_lst

lst = [1, 2, 2, 3, 4, 5, 6, 4, 1]
print(f"Updated list = {remove_duplicate(lst)}")
  2. Write a Python function to count the number of strings from a given list of strings. The string length is 2 or more and the first and last characters are the same.
    Sample List: ['abc', 'xyz', 'aba', '1221'] -> Expected Result: 2 
    def count_matching_strings(string_list):
    count = 0
    for s in string_list:
        if len(s) >= 2 and s[0] == s[-1]:
            count += 1
    return count

sample_list = ['abc', 'xyz', 'aba', '1221']
result = count_matching_strings(sample_list)
print(f"Sample List: {sample_list}")
print(f"Expected Result: {result}")
  3. Write a Python function to get a list, sorted in increasing order by the last element in each tuple from a given list of non-empty tuples.
    Sample List: [(2, 5), (1, 2), (4, 4), (2, 3), (2, 1)] -> Expected Result: [(2, 1), (1, 2), (2, 3), (4, 4), (2, 5)] 
    def sorted_lst(lst):
    for i in range(len(lst)-1):
        for j in range(len(lst)-1):
            if lst[j][1] > lst[j+1][1]:
                lst[j], lst[j+1] = lst[j+1], lst[j]

lst = [(2, 5), (1, 2), (4, 4), (2, 3), (2, 1)]
sorted_lst(lst)
print(lst)

  6. Write a Python program to check if each number is prime in a given list of numbers. Return True if all numbers are prime otherwise False.
    Sample Data: ([0, 3, 4, 7, 9]) -> False, ([3, 5, 7, 13]) -> True, ([1, 5, 3]) -> False 
    def is_prime(n):
    if n <= 1:
        return False
    for i in range(2, int(n**0.5) + 1):
        if n % i == 0:
            return False
    return True

def all_are_prime(num_list):
    for i in num_list:
        if not is_prime(i):
            return False
    return True

print(f"([0, 3, 4, 7, 9]) -> {all_are_prime([0, 3, 4, 7, 9])}")
print(f"([3, 5, 7, 13]) -> {all_are_prime([3, 5, 7, 13])}")
print(f"([1, 5, 3]) -> {all_are_prime([1, 5, 3])}")

Syntactical & Conceptual Questions

  1. State with proper reasoning which of the following are syntactically correct and which are syntactically incorrect function headers:
    • def ELSE(s=2, 7):
    • def else(a=2, b=3):
    • def Else(j=2, k):

    def ELSE(s=2, 7):
    Verdict: Syntactically Incorrect.
    Reasoning: A positional parameter (7) cannot follow a default argument (s=2).

    def else(a=2, b=3):
    Verdict: Syntactically Incorrect.
    Reasoning: else is a reserved keyword in Python.

    def Else(j=2, k):
    Verdict: Syntactically Incorrect.
    Reasoning: A positional parameter (k) cannot follow a default argument (j=2).

  2. State with proper reasoning which of the following function calls for the function fnc are correct and which are incorrect: 
    def fnc(bat, hat, vat=2, mat=4):
    return bat-vat+hat*mat
    • print(fnc(2, 5))
    • print(fnc(2, 4, 1, bat=4, mat=1))
    • print(fnc(vat=1, 3, 7))

    print(fnc(2, 5)):
    Verdict: Correct.
    Reasoning: Positional arguments are passed correctly, and vat, mat use default values.

    print(fnc(2, 4, 1, bat=4, mat=1)):
    Verdict: Incorrect.
    Reasoning: bat gets a value twice, which causes a TypeError.

    print(fnc(vat=1, 3, 7)):
    Verdict: Incorrect.
    Reasoning: Positional arguments (3, 7) cannot follow a keyword argument (vat=1).
